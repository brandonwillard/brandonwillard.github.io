Dynamic Linear Model Optimizations in Theano

We construct dynamic linear models (DLMs) in Theano and number of optimizations that facilitate the production of closed-form solutions.

Introduction

We start by considering the simple form of a DLM (Harrison &amp; West 1999) with prior \(\theta_0 \sim \operatorname{N}\left( m_0, C_0 \right)\):

\[\begin{align} y_t &= F_t^{\top} \theta_{t} + \epsilon_t, \quad \epsilon_t \sim \operatorname{N}\left( 0, V \right) \label{eq:basic-dlm-obs} \\ \theta_t &= G_t \theta_{t-1} + \nu_t, \quad \nu_t \sim \operatorname{N}\left( 0, W \right) \label{eq:basic-dlm-state} \end{align}\]

The most “notationally” faithful representation of this model in Theano is provided in Listing 2. It represents the generalized notion of a difference equation to the best of Theano’s abilities by using the scan operator. The scan operator codifies the right-hand sides of the difference equations in \(\eqref{eq:basic-dlm-obs}\) and \(\eqref{eq:basic-dlm-state}\) by constructing a Theano Function object taking values of \(t\), more or less.

In 4 we plot a sample from the model in Listing 2 for a fixed RNG seed.

Our goal is to automate some of the basic steps in the process of analytically manipulating and/or solving DLMs for the purpose of producing efficient and accurate posterior estimates. To that effect, we will want to compute as many closed-form operations as possible, and the prior predictive state and observation distributions are a good place to start.

Given all the prior and observed data up to time \(t\), \(D_t\), these distribution are given by the following:

\[\begin{align} \theta_{t} \mid D_{t-1} &\sim \operatorname{N}\left( a_{t}, R_{t} \right) \\ y_{t} \mid D_{t-1} &\sim \operatorname{N}\left( f_{t}, q_{t} \right) \end{align}\]

The prior predictive moments are as follows:

\[\begin{gather} a_t = G_t m_{t-1}, \quad R_t = G_t C_{t-1} G_t^\top + W_t \\ f_t = F_t^\top a_{t}, \quad Q_t = F_t^\top C_{t-1} F_t + V_t \end{gather}\]

We’ll also want to compute the posterior moments for \(\theta_t \mid D_t\), which are as follows:

\[\begin{gather} m_t = a_{t} + R_t F_t Q_t^{-1} \left(y_t - f_t\right), \quad C_t = R_t \left( I - F_t \left(R_t F_t Q_t^{-1}\right)^\top \right) \label{eq:dlm-post-moments} \end{gather}\]

These “filtered” moments/distributions are only one kind of posterior results for a DLM; they only take into account the data up to time \(t\). The other kind are the “smoothed” distributions, which provided posterior distributions for each time \(t\) given all past–and potentially future–observations. Notationally, we’ve used \(D_t\) to signify all conditional observations and parameters up to time \(t\), so the smoothed distributions are given by \(\theta_t \mid D_T\).

In most cases, models will not be as simple as the standard DLM. Even so, these basic closed-form solutions can still be relevant.

For instance, efficient MCMC algorithms can be constructed using these closed-form results for conditionally linear models. In those cases, we can compute the posterior moments–in closed-form–conditional on samples generated by other means. The standard approach is called forward-backward sampling and uses smoothed posteriors \(\theta_t \mid D_T\), which can also be computed in closed-form given the results above.

Forward-backward Estimation

We can use the smoothing and filtering steps in the previous section to perform a more efficient MCMC estimation than would otherwise be possible without the filtered and smoothed moments.

Forward-backward estimation (Fr-Schnatter 1994) works by first computing the forward filtered moments, allowing one to draw θ_T from \(\left(\theta_T \mid D_T\right) \sim \operatorname{N}\left(m_T, C_T\right)\) and, subsequently, \(\theta_t\) from \(\left(\theta_t \mid \theta_{t+1}, D_t \right) \sim \operatorname{N}\left(h_t, H_t\right)\).

The latter distribution’s moments are essentially a result of smoothing:

\[\begin{gather} B_t = C_t G^\top_{t+1} R_{t+1}^{-1} \\ h_t = m_t + B_t \left(\theta_{t+1} - a_{t+1}\right) \\ H_t = C_t - B_t R_{t+1} B^\top_t \end{gather}\]
